如何配置Windows terminal以优雅地使用Git Bash

2020-03-31

看看这个颜值，又有谁不爱？

  • Step 1 在Microsoft Store下载Terminal应用。安装完成后打开此应用则默认可以使用Windows Poweshell、 cmd、 你安装的wsl子系统； 但是默认是没有Git Bash的。

  • Step 2

    点击如图所示的Settings，进入配置文件编辑页面。

    在如图所示的位置插入如下描述。注意！不能忘记在上一个代码块后面加上半角逗号，否则会报错。

     {
  "acrylicOpacity": 0.5, // 透明度
  "closeOnExit": true, // 关闭的时候退出命令终端
  "commandline": "C:\\Program Files\\Git\\bin\\bash.exe", // gitbash的命令行所在位置
  "cursorColor": "#FFFFFF", // 光标颜色
  "cursorShape": "bar", // 这个，还没发现啥作用
  "fontSize": 14, // 终端字体大小
  "guid": "{1c4de342-38b7-51cf-b940-2309a097f589}", // 唯一的标识，这个记得改成和其他的终端不一样就行了，随便起一个
  "historySize": 9001, // emmm
  "icon": "C:\\Program Files\\Git\\git-icon.png", // git的图标，打开终端时候会看到
  "name": "Git Bash", // tab栏的标题显示
  "padding": "10, 10, 10, 10", // 边距
  "snapOnInput": true, // emmm
  "startingDirectory": "C:\\Users\\10536", // gitbash 启动的位置（默认在C盘的用户里面的就是 ~ ）
  "useAcrylic": true // 是否开启透明度
}

  • Step 3

    保存文件，甚至不用重启Terminal，你就可以使用颜值这样高的终端了。

解决VSCode使用gcc编译运行程序的乱码问题

2020-03-30

title: 解决VSCode使用gcc编译程序的乱码问题
date: 2020-03-04 20:23:20
tags:VSCode
我在前面有一篇幅博客之中介绍了如何配置文件使得VSCode这一款优秀的文本编辑器可以编译运行C\C++程序。然而不可忽视的问题是：使用f5调试程序会使得程序中文出现乱码。因为这个问题，我在遇到需要输出中文的时候我是一直使用的code runner插件。
直到今天，无意中发现了解决问题的方法。
Windows中文版的默认编码方式不是unicode而是GBK。当我们使用gcc编译程序时，gcc是没有那么只能的，终端输出的是同样的值在utf-8之中的字符。这是中文乱码出现的原因。而utf-8是大势所趋，我们所做的应该是将系统的默认编码改为utf-8。做法为：设置-语言-管理语言设置-管理-更改系统区域设置-把使用utf-8的方框勾选。

据我查到的资料，早期有一些程序员会通过获取本机的活动代码页也来判断用户所在地区，比如获得代码页为936，那对应的编码方式就是GBK，而unicode不存在什么地区的判定。

使用wsl2在Windows运行Linux系统

2020-02-17

  1. *”wsl” means “Windows Subsystem for Linux”.*

    wsl2 is the 2th generation of wsl , which has a much better performance.

  2. Linux[ˈlinʊks]

    此处引用陈学源同志告诉我的一句话:*”Linux is not Unix”*

  3. 使你的Windows10操作系统成为测试版（需要科学上网）

    1. 在右下角的通知栏打开设置
    2. 点击Windows预览版体验计划并加入
    3. 点击Windows更新-检查更新

    直接去微软官网下载易升获得的Windows10 1909版本尚不支持wsl2

  4. 在开始栏（小娜/控制面板）打开“启用或关闭Windows功能” 找到“适用于Windows的Linux子系统”选项并打勾，重启电脑

  5. 打开Microsoft Store，搜索Linux；建议下载Ubuntu；

  6. 下载完成后在开始栏找到它，并运行。等待几分钟后输入用户名，注意用户名的首字母不能大写，否则会报错；按下enter键以后输入密码，此时密码不可见。

  7. 以管理员权限运行powershell，键入命令wsl -l

    1. 此时如果出现wsl是外部指令，请返回步骤2；
    2. 如果能认wsl但是不认识-l，请返回步骤1；
    3. 如果没有出现你安装的Linux发行版系统，但出现了指令提示，请返回步骤3；

  8. 然后键入命令wsl --set-version <你安装的Linux发行版> 2

  9. 接着你可以在开始菜单中找到你安装的Linux发行版，运行，它现在已经可以完成独立系统可以完成的大部分操作了。但是它没有GUI，似乎是一个缺点。不过毕竟你已经拥有一个叫做Windows10的图形化操作系统了，还要什么自行车？接下来你可qian以wan尝bu试yao键入rm -rf/*命令。祝您成为一个优秀的Linuxer!

  10. return 0;
